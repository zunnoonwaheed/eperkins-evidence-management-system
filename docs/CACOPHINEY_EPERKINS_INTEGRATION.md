# Cacophiney Eperkins Certificate Integration

## Overview

Cacophiney video automation application now creates Eperkins certificates automatically after successful video generation and upload. This document describes the complete integration.

## Architecture

The integration follows the proven Eperkins-app pattern:

1. **Video Generation**: Historical lead data recreates form journey with Playwright
2. **Video Upload**: Video uploaded to Google Cloud Storage
3. **Certificate Creation**: Certificate API called AFTER upload success
4. **Result Handling**: Certificate data returned in response

### Integration Flow

```
Historical Lead Loaded
  ↓
Form Process Recreated (Playwright)
  ↓
Video Generated & Saved
  ↓
Video Uploaded to GCS → Final URL Received
  ↓
Certificate Payload Built from Real Lead Data
  ↓
Eperkins Certificate API Called
  ↓
cert_uuid and certificate_url Returned
  ↓
Result Included in Response
```

## Implementation Details

### Files Created

- `Cacophiney-app/eperkins_certificate_client.py` - API client with retry logic
- `Cacophiney-app/certificate_payload.py` - Payload builder for Cacophiney data
- `Cacophiney-app/test_certificate_integration.py` - Integration tests
- `Cacophiney-app/.env.example` - Environment variable template

### Files Modified

- `Cacophiney-app/app.py` - Added certificate creation after video upload
- `Cacophiney-app/requirements.txt` - Added `requests==2.31.0`

### Certificate Trigger Point

**Location**: `app.py` lines 381-392 (single) and 464-487 (bulk)

**Trigger condition**: After `save_video_link()` returns successfully

**Key behavior**:
- Video generation MUST complete successfully
- Video upload MUST complete successfully
- Final video URL MUST be available
- Only then is certificate API called

If certificate creation fails, video remains successful.

## Configuration

### Environment Variables

Add to `.env` or deployment environment:

```bash
# Eperkins Certificate Integration
EPERKINS_CERTIFICATE_API_URL=http://localhost:3000/api/certificates/create
EPERKINS_CERTIFICATE_API_KEY=<your-api-key>
EPERKINS_COMPANY_KEY=cacophiney
EPERKINS_WEBSITE=cacophiney.com
EPERKINS_SOURCE_SYSTEM=cacophiney-video-automation
```

**Production URL**:
```bash
EPERKINS_CERTIFICATE_API_URL=https://eperkinslaw.com/api/certificates/create
```

### Next.js Eperkins App

The Eperkins Next.js application must have the corresponding API key configured:

```bash
CERT_API_KEY_CACOPHINEY=<same-key-as-above>
```

## Certificate Payload Mapping

### Lead Data Fields

Real Cacophiney CSV columns → Certificate payload:

```python
{
  "company_key": "cacophiney",
  "website": "cacophiney.com",
  "source_system": "cacophiney-video-automation",
  "lead_id": "<auto-generated>",
  "lead_data": {
    "firstName": "First Name",
    "lastName": "Last Name",
    "email": "Email",
    "phone": "Mobile Phone",
    "form_answers": {
      "tax_debt_amount": "Tax Debt Amount",
      "contact_consent": "Contact Consent",
      "ip_address": "IP Address"
    }
  },
  "recording_id": "<auto-generated from filename>",
  "recording_url": "<final GCS signed URL>",
  "recording_storage_path": "gs://bucket/video.mp4",
  "lead_submitted_at": "Receipt Date (ISO-8601)",
  "video_generated_at": "<current UTC time>",
  "recording_type": "reconstructed_historical_recording",
  "idempotency_key": "cacophiney:<lead_id>:<recording_id>"
}
```

### Tax Debt Options

Cacophiney uses specific debt range options:
- "Under $10,000"
- "$10,000 – $24,999"
- "$25,000 – $49,999"
- "$50,000 – $99,999"
- "$100,000+"
- "Not sure"

## Idempotency

Format: `cacophiney:<lead_id>:<recording_id>`

**Behavior**:
- First request with this key creates certificate
- Subsequent requests return `duplicate: true` with same cert_uuid
- No duplicate certificates created
- Safe to retry failed requests

## Response Structure

### Success Response

```python
{
  "video": {
    "url": "https://storage.googleapis.com/.../video.mp4?...",
    "success": True
  },
  "certificate": {
    "success": True,
    "cert_uuid": "abc123...",
    "certificate_url": "https://eperkinslaw.com/certificates/abc123",
    "duplicate": False,
    "status": "generated"
  }
}
```

### Certificate Failure (Video Still Successful)

```python
{
  "video": {
    "url": "https://storage.googleapis.com/.../video.mp4?...",
    "success": True
  },
  "certificate": {
    "success": False,
    "error": "Connection failed",
    "warning": "Video completed, but certificate creation failed."
  }
}
```

## Error Handling

### API Client Retry Logic

**Retries on**:
- Connection failures
- Timeouts (30s)
- HTTP 429, 500, 502, 503, 504

**Does NOT retry on**:
- HTTP 400, 401, 403, 404
- Invalid payload
- Missing API key

**Maximum attempts**: 3 with exponential backoff

### Safe Logging

**Logged**:
- company_key
- lead_id
- recording_id
- cert_uuid
- HTTP status codes
- Error types

**Never logged**:
- API keys
- Full lead data
- Signed URL query parameters
- Authorization headers

## Troubleshooting

### Certificate creation fails with 401

**Cause**: API key mismatch

**Fix**: Ensure `EPERKINS_CERTIFICATE_API_KEY` matches `CERT_API_KEY_CACOPHINEY` in Eperkins app

### Certificate creation fails with 400

**Cause**: Invalid payload structure

**Fix**: Run `python test_certificate_integration.py` to verify payload mapping

### No certificate_url in response

**Cause**: Required field missing from lead data

**Fix**: Check that First Name, Last Name, Email, Phone are present in CSV

### Video URL is relative path

**Cause**: `APP_URL` environment variable not set

**Fix**: Set `APP_URL=http://localhost:5000` (or production domain)
